# AIMessage

`AIMessage` is the output type returned by all [[langchain-models|LangChain chat models]]. It represents the model's response and can contain text content, [[tool-call|tool calls]], reasoning blocks, and usage metadata.

## Content

### Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `text` | `string` | Convenience accessor for the text content |
| `content` | `string \| ContentBlock[]` | Raw content — string or list of content blocks |
| `contentBlocks` | `ContentBlock.Standard[]` | Lazily parsed standard blocks (see [[multimodal-content]]) |
| `tool_calls` | `ToolCall[]` | Tool calls the model wants to execute; empty array if none |
| `id` | `string` | Unique message ID (generated by LangChain or returned by provider) |
| `usage_metadata` | `UsageMetadata` | Token counts: `input_tokens`, `output_tokens`, `total_tokens`, `input_token_details`, `output_token_details` |
| `response_metadata` | `object` | Provider-specific response metadata (model name, finish reason, etc.) |

### Tool calls

When a model is bound with tools and decides to call one, the resulting `AIMessage` has `tool_calls` populated:

```typescript
const modelWithTools = model.bindTools([getWeather]);
const response = await modelWithTools.invoke("What's the weather in Paris?");

for (const toolCall of response.tool_calls) {
  console.log(toolCall.name);  // "get_weather"
  console.log(toolCall.args);  // { location: "Paris" }
  console.log(toolCall.id);    // "call_abc123" — must be echoed in ToolMessage
}
```

Each `ToolCall` has `name`, `args` (object), and `id`. The `id` must be passed as `tool_call_id` in the corresponding [[tool-message]].

### Token usage

```typescript
const response = await model.invoke("Hello!");
console.log(response.usage_metadata);
// {
//   input_tokens: 8,
//   output_tokens: 304,
//   total_tokens: 312,
//   input_token_details: { cache_read: 0 },
//   output_token_details: { reasoning: 256 }
// }
```

`output_token_details.reasoning` is populated by reasoning models (e.g., `o3`, Claude with extended thinking). Access via `UsageMetadata`.

### Manually constructing AIMessage

You can create an `AIMessage` by hand and insert it into the conversation history as if it came from the model. This is useful for priming few-shot examples or resuming an interrupted conversation:

```typescript
import { AIMessage, SystemMessage, HumanMessage } from "langchain";

const messages = [
  new SystemMessage("You are a helpful assistant"),
  new HumanMessage("Can you help me?"),
  new AIMessage("I'd be happy to help you with that question!"),
  new HumanMessage("Great! What's 2+2?"),
];
const response = await model.invoke(messages);
```

### Streaming: AIMessageChunk

During streaming, the model emits `AIMessageChunk` objects. Each chunk carries a partial `content` and partial `tool_calls`. Accumulate them with `.concat()`:

```typescript
import { AIMessageChunk } from "langchain";

let final: AIMessageChunk | undefined;
for await (const chunk of await model.stream("Explain recursion")) {
  final = final ? final.concat(chunk) : chunk;
}
// final is now equivalent to the full AIMessage
```

`AIMessageChunk` implements the same interface as `AIMessage` after concatenation.
